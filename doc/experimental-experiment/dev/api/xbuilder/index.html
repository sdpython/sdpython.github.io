Defines all the optimization to apply.

Parameters:

  • remove_unused â€“ remove all unused nodes, this must be true if pattern optimization is enabled

  • constant_folding â€“ folds constant as much as possible

  • constant_size â€“ all node Constant above this threshold should be defined as initializer

  • remove_identity â€“ remove identity nodes

  • patterns â€“ list of pattern optimization to apply to the graph, it looks a a specific subsequence of nodes in a graph and do some replacements, â€˜defaultâ€™ means a default list of optimization patterns are applied, see below for the most common values

  • constant_fusing â€“ similar node Constant and ConstantOfShape are used, this options avoids creating new nodes when they are the same

  • max_iter â€“ maximum number of iteration when doing pattern optimizations, -1 to let it undefined

  • recursive â€“ optimizes subgraphs and functions as well

It is possible to define a precise of the pattern to apply to a model. The value is interpreter by experimental_experiment.xoptim.get_pattern_list().

  • patterns="TransposeTranspose,TransposeMatMul": applies two patterns

  • patterns="default: applies all patterns modifying standard onnx operators into other standard onnx operators

  • patterns="default+onnxruntime: applies all patterns modifying standard onnx operators into other standard onnx operators as well as patterns fusing nodes into custom operators implemented by onnxruntime
